# S10-M2H: Selective Hybrid 重新验证分析

**日期**: 2026-05-03
**配置**: st=8, nt=16, ir=64, pl=4
**数据集**: SIFT10M UInt8 + DEFAULT

## 总结

**结果**: ❌ **失败** - M2-H 不能为 SIFT10M 重开

I/O wait 分布均匀，top 10% postings 只贡献 27.51% 的总 wait，远低于 40% 阈值。

| 指标 | 值 | 阈值 | 状态 |

| Top 1% postings wait 贡献 | 7.98% | N/A | - |

| **Top 10% postings wait 贡献** | **27.51%** | **>= 40%** | **❌** |
| Top 20% postings wait 贡献 | 41.97% | N/A | - |
| Gini 系数 | 0.34 | 越高越集中 | 低 |

## 详细分析

### I/O Wait 分布

| 百分位 | Posting 数量 | Wait 贡献 |

| Top 1% | 4,981 | 7.98% |
| Top 5% | 24,906 | 18.10% |

### 与 SIFT1M 对比

| 指标 | SIFT1M | SIFT10M | 结论 |
|------|--------|---------|------|
| Top 10% wait 贡献 | 29.63% | 27.51% | 相似，都均匀 |
| Gini 系数 | ~0.30 | 0.34 | 相似，集中度低 |
| M2-H 决策 | 已停止 | **已停止** | 一致 |

### 为什么 M2-H 在 SIFT10M 上失败

1. **I/O wait 均匀分布**: 没有哪个 posting 特别慢
2. **无热点**: 即使"最差"的 posting (363014) 也只有 45.5ms 总 wait
3. **Posting 数量大**: 10K 查询访问了 498,122 个不同的 posting
4. **Gini 系数低**: 0.34 表示接近均匀分布

### 最差的 5 个 Posting

| Posting ID | 总 Wait (ms) | 字节数 | 访问查询数 |
|------------|--------------|--------|------------|
| 363014 | 45.50 | 49,152 | 4 |
| 1366867 | 45.05 | 24,576 | 3 |
| 941071 | 44.85 | 24,576 | 3 |
| 173429 | 44.49 | 24,576 | 2 |
| 442242 | 44.40 | 36,864 | 3 |

"最差"的 posting 也只是平均值的 2-4 倍，不是异常值。

## 结论

M2-H selective hybrid **不能为 SIFT10M 重开**:

1. Top 10% wait 贡献 (27.51%) < 40% 阈值
2. 不存在集中的坏 posting 热点
3. Selective code-first 需要覆盖太多 posting
4. 与 SIFT1M 结论一致: I/O wait 均匀分布

## 对 Beyond Official Baseline Plan 的影响

M2-H 保持**已停止**状态。不需要实现 selective hybrid。
