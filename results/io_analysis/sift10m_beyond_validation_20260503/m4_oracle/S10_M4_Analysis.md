# S10-M4: Primary-Secondary 去重 Oracle 分析

**日期**: 2026-05-03
**配置**: st=8, nt=16, ir=64, pl=4
**数据集**: SIFT10M UInt8 + DEFAULT

## 总结

**结果**: ❌ **失败** - M4 不能为 SIFT10M 重开

查询级重复 payload 比例只有 10.91%，远低于 30% 阈值。

## 关键指标

| 指标 | 值 | 阈值 | 状态 |
|------|-----|------|------|
| 访问的 VID 总数 | 7,948,910 | - | - |
| 存在于多个 posting 的 VID | 81.16% | - | 高 |
| **查询级重复比例** | **10.91%** | **>= 30%** | **❌** |
| 存储级重复比例 | 49.72% | - | 误导性 |

## 关键洞察：存储级 vs 查询级重复

| 指标 | 存储级 | 查询级 |
|------|--------|--------|
| 重复字节数 | 1.94 GB (49.72%) | 0.43 GB (10.91%) |
| 解释 | VID 出现在多个 posting | 同一 VID 在同一查询中被多次读取 |

**49.72% 的存储级重复比例是误导性的！**

- 81.16% 的 VID 存在于多个 posting 中（存储重复）
- 但对于单个查询，大多数 VID 只从一个 posting 被访问
- 查询级重复只有 10.91%

## 详细分析

### VID 的 Posting 分布

| Posting 数量 | VID 数量 | 百分比 |
|--------------|----------|--------|
| 1 个 posting | 1,497,616 | 18.84% |
| 2 个 posting | 2,093,719 | 26.34% |
| 3 个 posting | 1,965,582 | 24.73% |
| 4 个 posting | 1,347,952 | 16.96% |
| 5+ 个 posting | 644,041 | 8.10% |

81% 的 VID 存在于多个 posting 中，但这不转化为查询级重复读取。

### 按查询的重复分布

| 百分位 | 重复比例 |

| P99 | 28.89% |

即使在 P99，重复比例 (28.89%) 也低于 30% 阈值。

### 与 SIFT1M 对比

| 指标 | SIFT1M | SIFT10M | 结论 |
|------|--------|---------|------|
| 查询级重复比例 | ~15% | 10.91% | 都低 |
| M4 决策 | 已停止 | **已停止** | 一致 |

## 为什么 M4 在 SIFT10M 上失败

1. **路由选择性**: 每个查询访问不同的 posting，即使同一 VID 存在于多个 posting
2. **Head 路由质量**: 良好的路由意味着每个查询只需要一小部分 posting
3. **无跨 posting 查询模式**: 查询通常不会通过多个 posting 访问同一 VID

## 数学解释

```
查询级重复 = (同一查询中从多个 posting 读取的 VID) / (总 VID 读取)
           = 3,225,339 / 29,555,670
           = 10.91%
```

320 万条去重记录是指**同一查询内**从多个 posting 访问同一 VID 的情况。

## 结论

M4 primary-secondary 去重**不能为 SIFT10M 重开**:

1. 查询级重复比例 (10.91%) << 30% 阈值
2. 存储级重复 (49.72%) 对查询优化是误导性的
3. Primary layout 不会显著减少查询页数
4. 与 SIFT1M 结论一致: 查询级重复不足

## 对 Beyond Official Baseline Plan 的影响

M4 保持**已停止**状态。不需要实现 primary-secondary 去重。

## 附录：存储级 vs 查询级重复

存储级重复和查询级重复的区别：

| 场景 | 存储重复？ | 查询重复？ |
|------|-----------|-----------|
| VID X 在 posting A 和 B 中 | ✓ 是 | ✗ 否（如果查询只读 A） |
| VID X 在同一查询中从 A 和 B 读取 | ✓ 是 | ✓ 是 |
| VID X 在查询 1 中读 A，查询 2 中读 B | ✓ 是 | ✗ 否（不同查询） |

M4 优化针对的是查询级重复，而这只有 10.91%。
